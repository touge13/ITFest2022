Добрый день, уважаемый пользователь! 

Базы данных, с которыми я работал при помощи PostgreSQL:
1. users - в этой таблице содержатся пользователи и их подписки для отправки им новых новостей;
2. news - в таблице содержатся только последние новости по подпискам;
3. webinfo - логин и пароль для авторизации на сайте
4. survey - тут содержатся отзывы пользователей
5. остальные таблицы, соответствующие подпискам - нужны для хранения последних пяти новостей,
на которые можно посмотреть, написав боту определенную команду.

В моем репозитории можно увидеть четыре основных файла:
1. config.py - в файле содержится (дополните самостоятельно):
	1. токен, благодаря которому можно подключиться к созданному чат-боту;
	2. токен, благодаря которому можно работать с vk api;
	3. пароль от базы данных.

2. parser.py - код, содержащийся в файле, обеспечивает постоянное обновление базы данных.
При появлении новой новости в вк, программа обновляет базу данных, добавляя прикрепленные к новости 
фото, видео, текст, id новости (вдальнейшем для распознавания новой записи), а также дату обновления бд 
новой записью (news и таблицу, соответствующую хэштегу в новой записи).
3. bot.py - в нем содержится код, который отвечает за функционал чат-бота и постоянную проерку на наличие
новой записи в таблице news(новой новости в группе ВКонтакте).
4. app.py - код отвечает за работоспособность сайта

Сейчас бот работает благодаря онлайн сервису Heroku (файлы runtime.txt, requirements.txt, Procfile на GitHub не загружены).
